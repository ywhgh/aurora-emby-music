## 0. 关键约束（适用于本计划所有改动）

### 0.1 敏感字段清单

以下字段**严禁**进入：仓库源码默认值、调试日志、错误信息字符串、诊断导出、上报通道。

| 字段 | 含义 |
|---|---|
| `serverUrl` | Emby 服务器地址 |
| `serverId` | Emby 服务器 ID |
| `userId` | Emby 用户 ID |
| `accessToken` | Emby 访问令牌 |
| `deviceId` / `deviceName` | 当前设备标识 / 名称 |
| `externalSourceApiUrl` | 音源桥 API 地址（用户自填） |
| `lyricsSourceBridgeApiUrl` | 歌词桥 API 地址（用户自填） |
| 任何出现在 `localStorage` / `IndexedDB` 中的外部 host | 包括用户配置的桥地址、Emby 服务器、企业内网代理等 |

### 0.2 强制脱敏

- UI 显示 server URL：保留 host 前两段 + 后两段，中间 `***`，端口打码。
- UI 显示 token：永远只显示 `***<last4>`。
- 任何 `console.log` / `setMessage` / 错误提示里如必须包含 URL，至少截断到 host。
- "复制诊断信息"功能在用户主动点击时可包含原值，但复制结果顶部必须加：
  > 本诊断包含你的服务器/账号信息，请勿在公开场合粘贴。

### 0.3 全局守则

- 不新增 npm 依赖（除非任务显式允许）。
- 每个 PR 单独可 `git revert`；禁止大爆炸重写。
- 不改 Emby 服务器契约（除非本计划中明确列出）。
- 不在 README / CHANGELOG 重复罗列修改；统一收口到 RELEASE_NOTES.md。

## 2. 任务清单

### 2.1 安全任务（S）

> 优先级：先做。每一项尽量独立成 PR。

#### S1 — source-bridge `/configure` 鉴权
- 新增 `--api-token` / `SOURCE_BRIDGE_API_TOKEN` 环境变量；POST 校验 `X-Bridge-Token`；缺失或不匹配 → 401。
- token 为空且未传 `--allow-anonymous-configure`（默认 false），启动时打显眼的 WARNING。
- 更新 `README.md` 的"音源桥"段。

#### S2 — `/remote-stream` SSRF 防护
- 新增 `BLOCKED_REMOTE_HOSTS`：localhost / 127.* / ::1 / 10.* / 172.16-31.* / 192.168.* / 169.254.* / 100.64-127.* / *.local / *.internal
- 接到 `url=` 后用 `dns.lookup` 解析；命中黑名单 → 403。
- 仅允许 http / https；拦截 `data:` / `javascript:` / `file:` / `gopher:` 等。
- CORS 改为受控白名单（默认拒绝跨域，除非 origin 在配置白名单内）。
- `scripts/source-bridge-smoke.js` 新增用例：127.0.0.1 / 192.168.x / 10.x 都必须 403。

#### S3 — 插件清单与插件代码签名校验
- manifest 新增 `signature` 字段；节点用 `crypto.verify` 校验。
- 公钥来自 `EMBY_BRIDGE_TRUSTED_KEYS` 环境变量（多个换行分隔）。
- 校验失败：整张 manifest 拒绝。
- 保留 `--allow-unsigned-plugins` 标志，默认 false；启动日志警告。
- 新增最小签名 fixture（`scripts/source-bridge-smoke.js`）。

#### S4 — `restorePluginTrackFromSnapshot` 的任意 pluginUrl 注入
- `?track=<JSON>` 中 `pluginUrl` 必须命中**当前内存中已加载**的插件；否则 404。
- 删除 `createRuntimePluginFromSnapshot` 的"按 URL 自动加载并执行"分支。
- 重启后只能通过 manifest 二次确认插件存在，否则视为失效。

#### S5 — 沙箱改用 `node:worker_threads`
- 用 `worker_threads` 替代 `vm.createContext`；每插件一 worker。
- 通信走 `parentPort.postMessage` 白名单方法：`search(query, page, type)`、`getMediaSource(raw, quality)`、`getLyric(raw)`。
- worker：CPU 3s 预算、`resourceLimits.maxOldGenerationSizeMb = 256`；超时硬 kill。
- worker 内禁访问 process / globalThis / 原生 require；只暴露 safe-fetch / safe-axios 注入包。

#### S6 — 加 CSP + Referrer-Policy
- `index.html` 加：
  ```html
  <meta http-equiv="Content-Security-Policy" content="default-src 'self'; img-src 'self' https: data: blob:; media-src 'self' https: http://localhost:* http://127.0.0.1:*; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self' https: http://localhost:* http://127.0.0.1:*; object-src 'none'; base-uri 'self'; frame-ancestors 'none';">
  <meta name="referrer" content="no-referrer">
  ```
- 联调：如有 SVG 图标被拦再放宽对应源；记录到 RELEASE_NOTES。

#### S7 — 移除 `src/config.js` 中硬编码的私有桥地址（敏感数据修复）
- 把 `DEFAULT_EMBY_LYRICS_SOURCE_BRIDGE_API_URL` 等用户私有配置的字面默认值**全部清空**。
- 改为动态来源：
  1. 启动从 `localStorage["emby-music-web/lyrics-source-bridge-api-url"]` 读取；
  2. 读不到返回空字符串，提示到设置页手动填；
  3. 不再回退到任何具体的公网 IP。
- 同一规则适用于 `EXTERNAL_SOURCE_API_KEY`、`SOURCE_BRIDGE_*_KEY` 等所有用户私有 key。
- `src/config.js` 中只保留公开常量（APP_NAME / APP_VERSION / POLICIES / LIMITS / PAGE_SIZE / SORT_KEYS 等）。

#### S8 — 诊断信息全量脱敏
- 新增 `src/redact.js`，提供：
  - `redactUrl(url)` → `https://us***@exa***.com:****`
  - `redactServer(server)` → `http://192.168.*.*:****/emby`
  - `redactToken(token)` → `***<last4>`
- app.js 的"复制诊断信息"按钮、`setMessage(...)`、所有 `console.log` 都需要过 redact helper。
- 如某些错误信息必须包含原始 URL 用于排查，至少截断到 host，不带端口 / path。
- README 新增"诊断信息安全"段。
- `scripts/smoke-check.js` 加 redact 三种输入的单元 case。

### 2.2 健壮性任务（R）

#### R1 — 清理未使用模块
- 删除：src/state-management.js、src/accessibility.js、src/color-extractor.js、src/performance.js、src/theme.js、src/ui-helpers.js（已 grep 确认 app.js / index.html / scripts 无引用）
- 更新 `index.html` 的 `<script>` 列表、`sw.js` 的 APP_SHELL、CHANGELOG。
- 注：color-extractor 与 theme 的能力在 F3 / F4 重新引入并接入，不直接照搬旧接口。

#### R2 — 统一安全 DOM helper
- 新增 `src/dom-helpers.js`：提供 `appendLoading(container, text)`、`appendEmpty(container, opts)`、`escapeHtml(x)`。
- 全文件扫描 `innerHTML = \`...${X}...\`` 模式（已知位置：app.js:7312、25419、25440、25444 等）：
  - 凡是来自 Emby / 插件 / 用户输入的 X，必须 `escapeHtml(X)`；
  - 静态 fragment（如 SVG icon）保留 innerHTML，但封装到 `setStaticMarkup(el, html)`，并加 lint 约束。
- 跑 `scripts/smoke-check.js` 确认无回归。

#### R3 — SW 升级流程改造
- sw.js install 不再 `skipWaiting()`；等收到 `SKIP_WAITING` 消息才激活。
- fetch 策略改为 stale-while-revalidate。
- 缓存名单（app shell）维持现状。

#### R4 — 队列持久化换 IndexedDB
- 新增 `src/idb-queue.js`，接口与 storage.js queue API 兼容。
- 上限 10000 条；保留 localStorage 作为降级（仅 80 条）。
- browser-smoke 覆盖：刷新后队列仍存在。

#### R5 — 拆 app.js
- 阶段化：
  - **R5a**（player.js、queue.js）
  - **R5b**（library.js、search.js）
  - **R5c**（settings.js、bridge.js）
- 入口改为 `main.js` 做 wiring；保持无构建（浏览器原生 ESM）。
- 每次抽完必须保证 `npm run check` + `npm run smoke:bridge` + `npm run smoke:browser` 全绿。

#### R6 — 状态层收敛
- 新增 `src/store.js`（小型 pub/sub，100 行内）。
- app.js 内 `state.x = y` 逐步替换为 `store.set({x: y})`；派生量（filteredTracks 等）改为按需计算 + `requestIdleCallback` 防抖。
- 不引入 React-like 框架。

### 2.3 功能任务（F）

#### F1 — 本地数据导入 / 导出
- 设置页"维护"区：导出（队列 / 收藏 / 最近 / 偏好）为 JSON；导入时校验 `version`，冲突前弹确认。
- 文件名 `aurora-export-<date>.json`，metadata 含 `appVersion`、`generatedAt`。
- 导入数据写入前先 redact 校验；禁止把导入的 serverUrl 当作可信 origin 重定向。

#### F2 — 歌词淡入淡出 + 卡拉 OK 高亮
- 现有同步歌词已成熟；加 120ms 过渡，主行 / 次行两级颜色梯度。
- verbatim（`<0.0>字`）解析已埋点，接入渲染（艺人字幕式高亮）。

#### F3 — 封面取色 → 主色 CSS 变量
- 新增 `src/cover-color.js`（不依赖 color-extractor.js 的旧 API）。
- 每次 currentTrack 变化时采样（要求 cover URL `crossOrigin="anonymous"`；不满足时降级为固定主题红）。
- 新增设置项"封面取色"开关；默认开。

#### F4 — 系统暗色模式跟随
- 设置项"主题：浅 / 深 / 跟随系统"。
- 监听 `matchMedia('(prefers-color-scheme: dark)')` change。

#### F5 — 键盘 Cheat Sheet UI
- 按 `?` 弹出当前所有快捷键列表（背景遮罩 + 居中卡片 + Esc 关闭）。
- 来源：app.js 已注册的 keyboardShortcut descriptors。

#### F6 — ReplayGain（轻量）
- 解析 `MediaSources[].MediaStreams[].ReplayGain`；存在则在 WebAudio 节点调 `gain.value = pow(10, rg/20)`。
- 设置项"启用 ReplayGain"；默认关。

#### F7 — 睡眠定时淡出
- 现版本硬切暂停；改为末 30 / 60 / 90 秒线性 fadeOut（设置项二选一）。

- 跑全量 smoke + browser-smoke（`BROWSER_SMOKE_RUN=1`）；

- 如果升版本，把 `package.json`、`src/config.js`、`sw.js`、`manifest.webmanifest`、`<link>` 的版本号统一同步。
