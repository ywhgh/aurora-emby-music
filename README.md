# Emby Music Web

一个响应式 Emby 音乐播放器网页原型，正在逐步完善成可日常使用的自托管音乐客户端：

- Emby 服务器地址规范化
- 登录地址兼容服务器根地址、`/emby` 路径和从 Emby Web 复制的 `/web/index.html` 地址
- 登录页不依赖浏览器 `url` 输入校验，避免地址格式被浏览器提前拦截而没有进入连接流程
- 登录页支持先测试 Emby 服务器连接，成功时显示服务器名和版本，方便区分地址问题和账号问题
- 登录页事件会优先绑定；如果应用内部初始化失败，会在登录页直接显示错误而不是按钮无响应
- 登录页显示当前版本，并提供复制诊断、清除应用缓存并重载的入口，便于排查旧 Service Worker 或旧脚本问题
- 登录页有内联兜底脚本；主脚本未加载时也能测试服务器、完成用户名密码认证、保存会话，并移除 URL 中的密码参数
- 如果地址栏里误带 `serverUrl`、`username`、`password` 参数，登录页会先清掉密码参数，再自动接管这次登录
- 主脚本也会二次读取并清理地址栏登录参数，避免兜底脚本被旧缓存或浏览器行为绕过后停在登录页
- “连接 Emby”同时监听按钮点击和表单提交，减少浏览器对表单事件处理差异造成的无响应
- 可在 `src/config.js` 里预填或锁定 Emby 服务器地址，适合固定服务器的自托管部署
- 用户名密码登录
- 本地保存 AccessToken、用户和服务器信息
- 刷新后自动校验会话
- 默认进入白色 Emby 登录页
- 登录成功后进入浅色、响应式的音乐客户端界面
- 移动端使用底部导航直达首页、音乐库、收藏、队列和设置，减少横向滚动导航负担
- 顶部工具区压缩为搜索、设置和头像菜单；刷新、连接测试、退出等低频操作收进账号弹窗，减少顶部拥挤
- 顶部和账号弹窗使用彩色线性 SVG 图标，保持轻量工具栏风格且不依赖外部图标资源
- 首页改为简约信息流：欢迎区、状态条、继续收听、歌单和歌曲入口减少卡片堆叠，底部播放器固定独立可见
- 移动端播放器改为紧凑 mini-player，保留曲目信息、播放键和进度，低频播放工具收进队列/设置入口，减少内容区被挤占
- 移动端底部导航提供“更多”入口，快速进入歌单、最近播放、专辑、艺人和正在播放
- “更多”入口使用通用操作菜单，次级页面会保持更多导航高亮
- 专辑、艺人和歌单详情页会高亮对应父级导航，减少详情页里的位置迷失
- 多音乐库服务器支持在音乐库页切换 Emby 音乐库分区，歌曲、专辑、艺人、收藏、搜索和加载更多会跟随当前分区，并按服务器用户记住上次选择
- 首页会按实际内容展示继续收听、我的歌单、收藏专辑和最近添加歌曲
- 首页会在存在本地恢复队列时显示继续播放卡片，支持从上次保存进度恢复或直接打开队列
- 恢复播放卡片会显示队列保存时间，例如刚刚保存、几分钟前保存，帮助确认恢复来源
- 支持 PWA manifest 和 service worker，可安装为独立窗口并缓存应用外壳
- PWA 检测到新版本时会在应用内提示，可一键切换并带版本参数重新载入，减少旧缓存导致的异常
- 支持浏览器在线/离线状态提示，网络恢复后可直接测试连接或刷新音乐库
- 支持 `#library`、`#nowPlaying`、`#settings` 等 hash 入口，便于桌面快捷方式直达常用页面
- 读取 Emby 音乐库、歌曲、专辑和艺人
- 支持搜索、专辑/艺人筛选、点击歌曲播放
- 搜索会先过滤已加载内容，并在后台向 Emby 服务端搜索更多匹配歌曲、专辑和艺人
- 支持专辑详情页、播放专辑、随机播放专辑
- 专辑详情页歌曲序号会优先使用 Emby 的碟号和曲号，多碟专辑显示为 `2-01` 这类格式
- 支持艺人详情页，按艺人查看专辑和歌曲，并可直接播放或随机播放
- 专辑、艺人和歌单详情页支持整组加入队列，不打断当前播放
- 专辑、艺人和歌单详情页支持整组下一首播放，会按原顺序插入当前曲目之后
- 歌曲列表里的艺人和专辑可直接跳转到对应详情；信息不足时自动按名称搜索
- 当前播放歌曲会在所有歌曲列表中用红色标题和播放/暂停标记显示，长列表里更容易定位
- 底部播放器支持一键定位当前歌曲，会优先在当前列表中定位，找不到时跳到播放队列
- 正在播放页的艺人和专辑也可直接跳转，方便从当前歌曲继续浏览
- 详情页返回按钮会回到进入详情前的页面，例如首页、收藏页、正在播放页或上一级详情
- 应用会记住各页面的滚动位置，从长列表进入详情再返回时不丢浏览位置
- 支持 Emby 歌单列表、创建空音频歌单和歌单详情，可播放歌单或随机播放歌单
- 歌曲行支持添加到已有 Emby 歌单，成功后会刷新本地歌单数量和当前歌单详情
- 歌单详情页支持上移、下移和拖拽调整歌曲顺序，并同步到 Emby 歌单
- 歌单详情页支持从当前歌单移除歌曲，只删除歌单条目，不删除媒体库文件
- 支持收藏/取消收藏歌曲、专辑、艺人和歌单，并 best-effort 同步到 Emby
- 收藏页会聚合展示收藏歌单、收藏专辑、收藏艺人和收藏歌曲
- 音乐库支持按 Emby `Genres` 风格、`ProductionYear` 年份和音质类型筛选，并和搜索、排序、批量播放一起工作
- 风格、年份、音质和收藏筛选会按服务器用户保存，刷新后继续保持整理状态
- 最近播放页会跟随搜索、风格、年份和音质筛选，和音乐库列表保持一致
- 筛选条会以 chip 展示搜索、专辑、艺人、风格、年份和音质条件，可单独移除某一项或一键清除全部
- 支持歌曲排序：最近添加、歌曲名、艺人、专辑、时长，并可选择默认、升序或降序；排序字段和方向会保存到本地
- 歌曲行会从 Emby `MediaSources` 显示轻量音频格式/码率徽标，专辑、艺人、歌单详情会汇总格式信息，并可按无损/有损快速筛选
- 底部播放器采用固定主控制区，左侧曲目信息、中间播放/进度、右侧队列/模式/音量始终可见，避免用户滚动查找播放按钮
- 底部播放器支持播放/暂停、上一首/下一首和进度跳转
- 底部播放器支持顺序、列表循环、单曲循环、随机模式
- 底部播放器和正在播放页支持睡眠定时，可从 15/30/45/60/90 分钟中选择，到点自动暂停并保留队列
- 设置页支持播放源策略：自动直连优先、仅直连、优先转码，并可选择转码码率
- 支持音量调节、静音和本地音量偏好保存
- 支持“正在播放”大播放页：大封面、歌曲信息、进度、控制、收藏和后续播放预览
- 正在播放页和底部播放器提供轻量播放波形，播放/暂停状态更直观，且不依赖跨域音频分析
- 当前播放歌曲会使用轻量确定性强调色联动封面占位、播放键、进度条、列表高亮和设置诊断状态，不依赖跨域封面取色
- 底部播放器和正在播放页会显示当前音频格式、直连/转码状态和播放源策略，便于排查播放问题
- 支持从 Emby 媒体附件/嵌入文本中读取 TXT/LRC 歌词，并在播放页同步高亮
- 支持键盘快捷操作：播放/暂停、切歌、快进快退、音量、静音、搜索聚焦、收藏当前歌曲、定位当前歌曲和打开队列
- 支持浏览器 Media Session：系统媒体键、锁屏/通知中心曲目信息和进度同步
- 浏览器标签页和 PWA 窗口标题会跟随当前播放歌曲更新，播放中会显示播放标记，便于任务栏切回
- 支持应用设置页：连接状态、浏览器网络、当前音乐库、排序状态、筛选状态、窗口标题、当前连接测试、PWA/媒体键状态、本地数据状态、快捷键、缓存清理和诊断信息复制
- 支持统一通知栏：关键失败场景提供重试、打开队列、查看设置、刷新等恢复入口
- 配置、格式化、歌词解析、Emby API helper 和本地存储 helper 已拆到 `src/`，保留无构建静态部署方式
- 支持刷新音乐库、清空搜索、随机播放和播放队列
- 音乐库支持按收藏状态筛选全部、已收藏和未收藏内容，方便整理资料库
- 音乐库支持舒适/紧凑列表密度切换，并保存到本地，长列表浏览可以显示更多歌曲
- 音乐库、专辑、艺人、歌单、收藏和队列等页面切换时会保留当前滚动位置，适合浏览大库
- 支持添加歌曲到队列、清空队列、队列内上移/下移/拖拽排序/移除
- 清空队列和清除已播后会提供撤销入口，降低误操作成本
- 播放队列页支持一键定位当前播放歌曲，长队列中更容易回到当前位置
- 播放队列页支持清除当前曲目前的已播歌曲，保留当前播放和后续待播队列
- 播放队列页支持整理当前曲目之后的待播顺序，按艺人、专辑、碟号、曲号和歌名恢复更自然的播放顺序
- 歌曲行支持“下一首播放”，可把歌曲插入当前曲目之后，已在队列中的歌曲会移动到下一首
- 移动端歌曲行会保留收藏和更多操作入口，把下一首、入队、加入歌单、排序和移除收进底部操作菜单
- 播放队列会显示总时长和当前曲目之后的剩余歌曲/剩余时长
- 播放队列支持随机重排当前曲目之后的待播歌曲，不改变当前播放和播放模式
- 音乐库、收藏歌曲和最近播放页支持播放全部和整页加入队列
- 音乐库、收藏歌曲、最近播放、播放队列和详情页会显示已加载歌曲总时长，方便决定是否整组播放
- 支持本地恢复播放队列和当前歌曲，刷新页面或重新打开 PWA 后不会丢队列
- 恢复队列时会显示上次保存的播放进度，播放按钮会从该位置继续
- 播放时会预加载下一首，降低连续听歌的切换等待
- 自动策略下直连播放失败时会尝试 Emby universal/transcode 播放地址，播放失败提示可手动切换直连/转码重试
- 支持歌曲、专辑、艺人分页加载更多
- 滚动接近底部时会自动加载更多歌曲、专辑、艺人和收藏
- 收藏同步失败时会回滚 UI，避免本地状态和服务器状态长期不一致
- 支持本地最近播放记录和清空记录，清空后可撤销恢复
- 播放前会请求 Emby `PlaybackInfo` 获取服务器认可的 `PlaySessionId`，并向 Emby 上报播放开始、播放进度和播放停止

## 运行

直接打开 `index.html` 即可预览界面。

如果浏览器或 Emby CORS 策略阻止 `file://` 发起请求，可以在本目录启动一个静态服务：

```powershell
python -m http.server 5173
```

然后访问：

```text
http://localhost:5173
```

## 音源桥

登录页有两种连接方式：

- `Emby`：连接 Emby 服务器。
- `音源桥`：可选入口，默认不用填写。它用于接本地音乐索引服务，或你自己实现的其他音源服务。

本地音乐可以启动内置桥接服务：

```powershell
npm run bridge -- --music-dir "D:\Music"
```

然后在登录页切到 `音源桥`，填写：

```text
http://127.0.0.1:5174
```

桥接服务会提供：

```text
GET /health
GET /tracks?offset=0&limit=100
GET /search?q=keyword&offset=0&limit=100
GET /media?id=trackId
GET /lyric?id=trackId
```

如果你有类似 `https://13413.kstore.vip/yuanli/yuanli.json` 的音源清单链接，不能直接填到网页里。这个链接是插件清单，不是播放器可直接消费的 HTTP API。可以先让桥接服务读取清单：

```powershell
npm run bridge -- --manifest-url "https://13413.kstore.vip/yuanli/yuanli.json"
```

再访问 `http://127.0.0.1:5174/sources` 查看识别结果。桥接服务不会直接执行远程插件 JS；如果要让这类清单可播放，需要在后端实现合法的适配器，把它转换成上面的 `/tracks`、`/search`、`/media` 标准接口。

## 本地检查

项目不需要安装运行依赖；需要 Node.js 来执行本地检查脚本：

```powershell
npm run check
```

该命令会检查应用版本和静态资源版本是否一致、CSS 结构是否完整、双语歌词关键解析是否正确，并对主要 JavaScript 文件做语法检查。

## 部署配置

如果这个网页只给一个固定 Emby 服务器使用，可以在 `src/config.js` 中配置：

```js
DEFAULT_SERVER_URL: "http://192.168.1.10:8096",
LOCK_SERVER_URL: true,
```

`DEFAULT_SERVER_URL` 会预填登录页服务器地址；`LOCK_SERVER_URL` 为 `true` 时，登录页会锁定服务器输入框，并且丢弃其它服务器的旧会话。

## 已知限制

- 当前还是静态无构建版本；`app.js` 负责主流程，`src/config.js`、`src/format.js`、`src/lyrics.js`、`src/emby-api.js`、`src/storage.js` 承接配置、工具、歌词、Emby 请求和本地持久化逻辑。
- PWA 只缓存应用外壳，不缓存 Emby 接口、封面和音频流；应用外壳资源带版本参数并优先走网络，减少旧缓存导致登录页不更新。
- 设置页的“测试当前连接”会重新验证服务器信息和当前用户 token，适合排查登录后加载或播放异常。
- 搜索会后台请求 Emby 扩展当前结果；仍受服务器权限、网络和 Emby 搜索能力影响。
- universal/transcode 依赖 Emby 服务器的转码能力和浏览器音频支持。
- 歌词依赖 Emby 对音频文件的歌词/文本附件识别；未读取到时会显示空状态。
- Media Session 能力取决于浏览器支持；不支持时会自动降级为网页内控制。
- 播放状态上报是 best-effort，失败不会打断本地播放。
- 最近播放和播放队列目前保存在浏览器本地；跨设备同步后续可继续接 Emby 播放历史。
