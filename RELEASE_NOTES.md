## 0.93.149

### 版本说明

本版本继续优化逐字歌词体验，并加入紧急播放稳定性修复：针对部分歌词源会返回重复或过密逐字时间戳的问题，播放器会在渲染逐字分组时自动铺开最小时间间隔，避免多个字在同一瞬间一起跳完；同时禁用沉浸页波形可视化对共享 audio 元素的 Web Audio 接管，避免部分浏览器出现有进度但无声音的情况。

### 更新内容

- 逐字歌词时间轴新增最小间隔正规化，重复时间戳会自动按 80ms 递进展开。
- 保留原有 `clip-path` 高亮热路径，不改成逐帧宽度布局写入，避免移动端出现新的绘制压力。
- 同一套正规化逻辑覆盖沉浸歌词、主歌词列表和当前播放歌词焦点。
- 紧急修复沉浸页波形可视化接管 `<audio>` 输出链导致部分浏览器播放无声的问题。
- 波形可视化暂时使用时间驱动的安全兜底，不再创建 `MediaElementSource`，优先保证音乐播放声音稳定。
- 增加真实 Chrome 冒烟测试场景，覆盖多个逐字时间戳完全相同的歌词，确认不会瞬间跳到后续字。

### 验证

```powershell
npm run check:js
npm run smoke
npm run smoke:bridge
$env:BROWSER_SMOKE_RUN='1'; $env:BROWSER_SMOKE_URL='http://127.0.0.1:5173/'; npm run smoke:browser
```

## 0.93.148

### 版本说明

本版本聚焦移动端沉浸播放器的中间视觉区：彻底移除旧的柱状律动条，改为更宽、更明显的横向流动波形。波形优先使用 Web Audio API 的时域采样跟随真实音乐振幅，取不到音频数据时才使用安静的流动兜底，整体保持暖白色音乐氛围，不再保留偏红柱状条的割裂感。

### 更新内容

- 移动端沉浸播放器默认封面区改为 SVG 横向波形，包含主线、柔光层、填充雾面和从左到右的流动高光。
- Web Audio 可视化从频谱柱改为 `getByteTimeDomainData()` 时域波形采样，律动更接近真实播放状态。
- 暂停或未播放时回落到低幅度安静波形，避免中间区域变成僵硬直线。
- 清理移动端旧 `.immersive-visualizer` 柱状条结构和样式，避免新旧视觉方案同时残留。
- 保留手机沉浸页点击中间区域切换封面/歌词的原逻辑，同时同步移动端收藏、纯享、音质、下载和更多操作入口。
- 增强真实 Chrome 冒烟测试，要求手机端波形足够宽高、使用平滑曲线路径，并确认旧柱状条 DOM 数量为 0。

### 验证

```powershell
npm run check:js
npm run smoke
npm run smoke:bridge
$env:BROWSER_SMOKE_RUN='1'; $env:BROWSER_SMOKE_URL='http://127.0.0.1:5173/'; npm run smoke:browser
```

## 0.93.147

### 版本说明

本版本聚焦播放稳定性和逐字歌词流畅度：优化 RAF 逐字进度使用的高精度歌词时钟，轻微音频时间漂移会渐进校正，避免 timeupdate 粗粒度同步造成逐字填充突然跳动；同时强化音乐桥缓存落盘和重新进入应用后的外部音源恢复链路，减少重新搜索才能播放的问题。

### 更新内容

- 逐字歌词时钟新增软校正机制，小幅漂移按比例拉回，不再每次都硬重置播放基准。
- 增加硬重同步阈值，seek、卡顿或真实大漂移时仍会快速追上音频时间。
- 音乐桥插件缓存位于临时目录时会立即落盘，降低重进应用后外部音源缓存丢失的概率。
- 沉浸式播放右上工具补齐音质、下载当前音乐和更多操作入口，下载会优先解析当前可用播放源。
- 沉浸式皮肤按钮文案统一为“皮肤样式”，线性图标和沉浸式图标加粗，提高鼠标经过和按钮识别度。
- 扩展静态、音乐桥和真实 Chrome 冒烟测试，覆盖软漂移校正、避免硬跳、硬漂移重同步、音乐桥缓存落盘和重进播放场景。
