# v0.93.85 发布说明

本版本重点修复音乐桥搜索结果的真实音质显示，并同步完善外部音源、本地音乐和视频结果的质量识别链路。

## 更新内容

- 搜索结果不再根据平台名称猜测 `SQ`、`MP3` 或 `1080P`，避免把可能音质误显示成真实音质。
- 音乐桥搜索结果会在后台解析 `/media`，解析完成后再显示真实返回的音质，例如 `FLAC`、`SQ`、`320k`、`MV 720P`。
- 解析中会显示 `检测中`，源站没有返回音质字段时显示 `源站未标注` 或 `MV 未标注`。
- 本地音乐通过音乐桥播放时会返回真实文件格式；如果环境存在 `ffprobe`，会进一步读取真实 codec 和 bitrate。
- 视频结果保留独立清晰度策略，播放或搜索解析后显示实际返回的视频质量。
- 修复搜索结果合并时旧对象覆盖新音质字段的问题。
- 更新缓存版本到 `0.93.85`，减少浏览器和 Service Worker 继续使用旧脚本的情况。

## 验证

```powershell
npm run check
```

## 使用提示

- 音乐桥相关更新需要重启 `npm run bridge` 后生效。
- 刷新网页建议使用：`http://localhost:5173/?v=0.93.85&reload=real-quality`
