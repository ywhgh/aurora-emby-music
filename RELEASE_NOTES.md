## 版本说明

本版本继续加固音乐桥恢复播放稳定性。重新进入软件后，如果队列中保存的是旧音乐桥生成的 `/plugin-stream` 或 `/remote-stream` 播放地址，前端不会再直接使用旧地址，而是通过当前音乐桥重新请求 `/media` 获取新的可播放链接。

## 更新内容

- 外部音源 inline URL 如果指向旧的 `/plugin-stream`，会强制走当前音乐桥重新解析。
- 外部音源 inline URL 如果指向旧的 `/remote-stream`，同样会绕过旧桥代理地址。
- 保留普通真实直链的默认行为，不会无故重新解析非音乐桥 URL。
- 增加外部音源回归检查，验证旧桥地址会改走当前桥 `/media`。
